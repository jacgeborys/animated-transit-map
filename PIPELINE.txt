Warsaw Transit Visualization - Pipeline
========================================

NEW PIPELINE (Simplified & Better):

1. Data Acquisition
   core/gtfs_downloader.py
   → Downloads GTFS data

2. Data Parsing
   core/gtfs_parser.py
   → Parses GTFS into usable format
   
3. Geometry Building
   core/route_line_builder.py
   → Creates route_lines_continuous.shp

4. Segment Creation ⭐ NEW!
   core/junction_segmenter.py
   → Creates junction_segments.shp
   → Creates junction_points.shp (debug file)
   → Can be previewed in QGIS before animation
   → Junction-based (natural convergence/divergence)
   → Progress bars show processing status

5. Animation/Visualization
   animate/animate_full_density.py (all routes, optimized)
   animate/animate_segments_density.py (5 specific tram lines)
   
   Both load pre-computed junction_segments.shp


KEY FILES:
----------
data/processed/route_lines_continuous.shp  (input for segmentation)
data/processed/junction_segments.shp       (output, preview in QGIS!)
data/processed/junction_points.shp         (debug: junction locations in QGIS!)
data/processed/schedule_for_animation.csv  (schedule data)

WHY THIS IS BETTER:
-------------------
✓ Segments computed once, reused many times
✓ Can debug/preview segments in QGIS
✓ Cleaner separation of concerns
✓ Faster animations (no re-computation)
✓ Junction-based segments match transit reality
